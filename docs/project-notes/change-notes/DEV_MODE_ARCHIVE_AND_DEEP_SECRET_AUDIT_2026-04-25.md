## 三、深度敏感信息核查结果

本轮采用了**文件名巡检、关键模式全文扫描、核心源码二次聚焦排查**三层方式。检查中未发现真实 API Key、访问令牌、私钥或可直接利用的 Bearer 凭据。

### 1. 核查发现

在排除历史扫描产物和旧清单噪声后，核心源码中的命中项仅剩两类：一类是认证接口中正常存在的 `password`、`currentPassword`、`newPassword` 等字段名；另一类是 QQ 通道解析 webhook 时的 `access_token` 变量名。这些命中均属于**字段或变量名**，并非真实泄漏值。

### 2. 已完成加固

对 `scripts/take_screenshots.py` 做了额外加固，以消除这轮深度检查中唯一值得整改的高风险实现。

| 加固项 | 调整结果 |
| --- | --- |
| 默认截图地址 | 从开发端口改为正式入口 `http://localhost:38000/` |
| 默认密码 | 不再内置 `admin` / `admin1234`，必须通过环境变量显式提供 |
| 新密码逻辑 | 仅在显式提供 `SCREENSHOT_NEW_PASSWORD` 时参与尝试 |
| 日志输出 | 不再打印当前使用的密码标记 |
