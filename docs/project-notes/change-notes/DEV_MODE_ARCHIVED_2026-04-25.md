# 开发模式归档说明

## 一、调整结论

自本次整理起，仓库的**当前主链路**不再保留开发模式作为公开部署与使用入口。正式使用路径统一收口为：运行 `./install.sh` 完成安装，再通过 `./agentorchestrator.sh start` 启动服务，并直接访问 `http://127.0.0.1:38000/`。

## 二、归档范围

此前文档中出现的以下内容，现统一视为**历史开发记录**，仅保留在项目归档目录，不再作为面向使用者的操作说明：

| 项目 | 处理方式 |
| --- | --- |
| `pnpm dev` / `npm run dev` 启动说明 | 移出主链路文档 |
| `http://127.0.0.1:35173` 等开发端口 | 不再作为公开入口 |
| “联调前端”“开发模式”“Vite 本地入口” 等表述 | 改写为归档说明或历史记录 |

## 三、保留原因

这些内容并非完全删除，而是转入归档，原因在于它们仍然能够解释历史端口变更、旧版联调策略以及此前排障过程中的决策依据。保留归档可以帮助后续审计和追溯，但不会继续干扰正式部署口径。

## 四、当前有效口径

| 分类 | 当前有效入口 |
| --- | --- |
| 安装入口 | `./install.sh` |
| 启动入口 | `./agentorchestrator.sh start` |
| 正式页面 | `http://127.0.0.1:38000/` |
| API 健康检查 | `http://127.0.0.1:38000/health` |

## 五、说明

后续若确需恢复独立开发模式，应该以新的专项设计说明重新引入，而不是继续把历史开发入口混入当前主链路文档。
