# README 工作流程图检查记录

## 当前检查对象

- `docs/readme-governance-flow.png`

## 观察结论

当前治理流程图虽然已经不是单一超长总图，但**仍然存在横向过宽、节点过多、分支线条过长、标签字号偏小**的问题。

图中主要可读性问题如下：

| 问题 | 表现 |
| --- | --- |
| 横向跨度过大 | 调度中心同时扇出 5 个 specialist，导致整张图被拉得很宽 |
| 线条交叉感明显 | 评审回退、调度扇出、归档汇聚同时出现，视觉负担较重 |
| 节点文字偏密 | 中英双语节点叠加后，在 README 中缩放显示时阅读压力较大 |
| 中层结构过于集中 | 规划、评审、调度三层都压在中心狭窄区域，纵向间距不足 |

## 对下一步重构的启示

建议把“总体治理链路”进一步拆成**多个更短的子流程图**，至少将下列内容分开表达：

1. **任务进入与治理主链路**：用户、任务入口、总控、规划、评审、调度。
2. **评审回退机制**：规划与评审之间的往返修订关系。
3. **调度与专业执行汇聚**：调度中心、各 specialist、结果归档、最终交付。

这样可以显著减少单图中的横向扇出和长回路线，使 README 中嵌入后的图更易读。
