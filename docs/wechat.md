## 这里保留什么内容？

### 架构拆解系列

这套多 Agent 协作架构不是空谈 metaphor，而是可运行、可复盘、可持续演进的工程系统。这里保留的是适合公开仓库直接阅读的实践主题概览：

- **为什么评审中心是杀手锏** —— 以及它拦下了多少次不成熟方案
- **为什么总控中心负责分拣** —— Intent Detection 在任务入口上的真实作用
- **为什么专业执行组可以并行不打架** —— 从任务解耦到 goroutine 的并发智慧
- **SOUL.md 人设怎么写** —— 让 AI 入戏的 prompt 炼丹术

### 踩坑复盘

做 12 个 Agent 协作，每天都有惊喜（惊吓）：

- ✍️ *Agent A 给 Agent B 发消息，B 说"朕不认识你"*
- 🔥 *Token 烧了 $50，结果评审中心把方案驳回了三次*
- 🐛 *调度中心派发任务，执行组集体罢工——原来 agent ID 写错了*
- 💀 *安装脚本接入现有环境时覆盖了用户自定义角色配置——后来改成先由 AI 做环境检查再执行*

这些都是真实 Issue，文章里逐条复盘怎么修的。

### Token 优化与成本控制

大家最关心的问题：**这玩意儿 token 不得烧穿？**

项目文档会持续补充：
- 用 Claude 内置 cache 砍掉 80% SOUL.md 重复读取
- 「快速通道」机制：简单任务跳过评审中心审核
- 按需启动 Agent，不轮询不烧钱
- 精简 SOUL.md：1000 字以内搞定一个高质量角色
