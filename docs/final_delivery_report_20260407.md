### 4. 上下文窗口管理机制

本轮新增了完整的**上下文窗口管理**能力，而不是仅在文档中提出建议。后端派发工作器现在会在派发前执行上下文容量评估，并输出结构化状态。

| 状态级别 | 含义 | 处理策略 |
|---|---|---|
| `normal` | 容量安全 | 正常拼装上下文并继续执行 |
| `warning` | 接近阈值 | 在任务元数据和前端中提示接近上限 |
| `critical` | 风险较高 | 优先压缩历史上下文、保留关键摘要与最近活动 |
| `overflow` | 超限风险 | 执行归档、缩写、续写提示和降级拼装 |

当前已落地的能力包括：

1. **容量预警**：根据配置阈值评估上下文占用，并将状态写入任务元数据。
2. **压缩归档**：在必要时裁剪长上下文，将历史流转、进展和冗长内容归档到文件路径中。
3. **续写衔接**：在归档后自动生成续写提示，帮助后续 Agent 继续接力而不丢失背景。
4. **前端可见化**：运行看板任务卡片与任务详情现在可显示“接近上限”“已压缩”“可续写”等提示，以及归档路径、压缩条数与建议动作。
