随后又对高频术语进行了关键词扫描，剩余命中主要分为两类：

1. **源码内部字段名或接口字段**，例如 `totalTokens` 一类的结构字段；

## 第五轮补充调整（2026-04-10）

本轮针对“全网搜索里的搜索记录入口不明显、用户难以找到”的反馈进行了补修，不再仅强调已有能力，而是直接调整了可见层级与默认呈现方式。现在在 `Web Search` 页面中，搜索记录区块被固定放置在**搜索输入框正下方**，并位于结果统计与结果列表之前，用户进入页面后即可第一时间看到该入口。

同时，搜索记录卡片补充了更清晰的说明结构。卡片内包含搜索记录标题、状态标签、说明文案、空状态提示、最近搜索词列表区域、单条恢复入口、单条删除入口，以及 `Clear History` 清空历史按钮。这样用户既可以一键恢复最近搜索，也可以按条清理历史记录，避免入口被高级筛选或结果内容淹没。
